# 成交状态管理功能实现说明

## 已实现的功能

### 1. 成交状态管理弹窗 (DealStatusModal)
- **位置**: `frontend/src/components/orderer/modals/DealStatusModal.jsx`
- **功能**:
  - 支持四种成交状态：待确认(默认)、谈单中、已成交、未成交
  - 成交金额支持小数，必须>0
  - 支持美元($)和人民币(¥)两种货币
  - 权限控制：仅下单人可编辑，管理员/负责人只读，接单人可见金额
  - 超期警告：超过3天未更新会显示警告

### 2. 成交看板 (DealBoard)
- **位置**: `frontend/src/components/orderer/DealBoard.jsx`
- **功能**:
  - 按成交状态分组显示：待确认、谈单中、已成交、未成交
  - 超过3天未更新的订单高亮显示（红色边框+警告图标）
  - 显示成交金额（用于接单人KPI统计）
  - 点击订单卡片可打开成交状态管理弹窗

### 3. 订单表格增强 (OrderTable)
- **位置**: `frontend/src/components/orderer/OrderTable.jsx`
- **新增列**:
  - 成交状态列：仅已完成/已拒绝订单显示，点击可管理
  - 成交金额列：显示货币符号和金额

### 4. 通知中心 (NotificationCenter)
- **位置**: `frontend/src/components/orderer/NotificationCenter.jsx`
- **功能**:
  - 仅在订单状态为"已成交"时触发通知
  - 通知内容简短：订单编号 + 成交金额
  - 红色脚标提醒未读通知数量
  - 关闭弹窗后自动标记所有通知为已读

### 5. 枚举常量更新 (enums.js)
- **位置**: `frontend/src/constants/enums.js`
- **新增**:
  - ORDER_STATUS 增加 7-10 状态码
  - DEAL_STATUS 独立枚举：待确认/谈单中/已成交/未成交

### 6. 视图切换
- **功能**:
  - 表格视图：传统列表展示
  - 看板视图：按成交状态分组的卡片展示
  - 通过按钮组切换

## 数据字段说明

订单对象新增字段：
```javascript
{
  deal_status: 7,           // 成交状态：7-待确认, 8-谈单中, 9-已成交, 10-未成交
  deal_amount: 5800.00,     // 成交金额（必须>0，支持2位小数）
  currency: 'CNY',          // 货币：'CNY' 或 'USD'
  deal_updated_at: '...',   // 最后更新时间
  deal_notified: false      // 是否已通知
}
```

## 权限说明

| 角色 | 查看成交状态 | 修改成交状态 | 查看成交金额 |
|------|------------|------------|------------|
| 下单人 | ✓ | ✓ | ✓ |
| 接单人 | ✓ | ✗ | ✓ (KPI统计) |
| 管理员/负责人 | ✓ | ✗ | ✓ |

## 业务规则

1. **默认状态**: 订单完成时，成交状态默认为"待确认"(7)
2. **金额验证**: 选择"已成交"时，必须填写大于0的成交金额
3. **超期提醒**: 订单完成后3天内需更新成交状态，超过3天高亮显示
4. **通知触发**: 仅在状态变更为"已成交"时触发通知
5. **通知内容**: 简化为关键信息 - "订单 xxx 已成交，成交金额 ¥xxxx.xx"

## 使用方式

### 下单人视角
1. 在订单列表中，点击已完成订单的"成交状态"标签
2. 在弹窗中修改成交状态和金额
3. 切换到"看板视图"查看所有订单的成交情况

### 接单人视角
1. 可查看自己接单的成交金额（用于KPI统计）
2. 无法修改成交状态
3. 可在通知中心看到成交通知

### 管理员/负责人视角
1. 可查看所有订单的成交状态和金额
2. 无法修改成交状态
3. 可通过看板视图快速统计成交情况

## 测试建议

1. **测试成交状态修改**:
   - 以下单人身份登录
   - 找到已完成的订单
   - 点击成交状态标签，修改为不同状态
   - 验证金额验证逻辑

2. **测试看板视图**:
   - 切换到看板视图
   - 验证四个分组的订单分布
   - 检查超期订单的高亮显示

3. **测试通知中心**:
   - 将订单状态修改为"已成交"
   - 检查通知中心是否出现新通知
   - 验证通知内容格式
   - 关闭弹窗后检查脚标是否消失

4. **测试权限控制**:
   - 分别以不同角色登录
   - 验证编辑权限是否正确
   - 验证金额可见性
